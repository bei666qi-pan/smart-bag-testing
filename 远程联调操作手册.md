# 智能书包 远程联调操作手册（一次 USB 烧录 → 之后全程无线）

目标：ESP32-S3 只插一次线烧录带 OTA 的固件；之后改完代码 → 无线推固件 → 远程看日志 →
MQTT 驱动全链路联调，全程不用再插数据线。

---

## 0. 工作原理（先理解这一点）

ESP32-S3 **不跑 Linux、没有 SSH**，它跑的是一整个编译好的 C++ 固件镜像。所以"远程改代码"
的真实形态是：**在电脑/Jetson 上改代码 → 编译 → 通过 WiFi 把新固件整包推上去（OTA）**，
而不是登录设备在线编辑。本固件已内置两件远程联调利器：

- **ArduinoOTA（无线烧录）**：设备联网后以 `smart-bag-esp32.local` 出现，可用 WiFi 推新固件。
- **telnet 远程日志（端口 23）**：把 MQTT 连接、每条命令、每个 ACK、周期传感器/GPS 上报
  实时推到远程，`telnet <ip> 23` 即可看，替代插线看 USB 串口。

> 配置项在 `esp32代码/Hardware/OTAUpdate.h`：
> `OTA_HOSTNAME="smart-bag-esp32"`、`OTA_PASSWORD="<OTA_PASSWORD>"`（可改/置空）。

## 1. 第一次：USB 线烧录（只此一次）

用 Arduino IDE（或 arduino-cli），插 USB：

1. 开发板选 **ESP32S3 Dev Module**。
2. **分区方案必须带 OTA**（关键，否则无法无线烧录）：选
   **"Minimal SPIFFS (1.9MB APP with OTA / ...)"** 或其它含两个 app 分区（ota_0/ota_1）的方案。
   本工程带摄像头+多个库，固件较大，务必选 APP 空间足够且含 OTA 的方案。
3. 打开 PSRAM：**PSRAM: "OPI PSRAM"**（摄像头需要）。
4. 编译并经 USB 上传。
5. 打开串口监视器（115200），确认看到：
   - `WiFi 已连接，IP: 192.168.x.x`
   - `[telnet] 远程日志已启动: telnet 192.168.x.x 23`
   - `[OTA] 就绪: 主机名 smart-bag-esp32.local, IP 192.168.x.x`
   - `=== 远程联调服务已就绪（OTA + telnet）===`

记下这个 IP（下面叫 `<ESP_IP>`）。至此 USB 线可以拔了。

---

## 2. 之后：无线烧录新固件（OTA）

电脑/Jetson 与 ESP32 在**同一热点/局域网**即可。按"谁来刷"选一种，不用都看：

### 你自己手动刷 → Arduino IDE 图形界面（首选）

工具 → 端口 → **网络端口**里会出现 `smart-bag-esp32`，选它 → 点上传 → 输入口令 `<OTA_PASSWORD>`。
全程点鼠标，不碰命令行。

### 交给 AI 在 Jetson 上自动刷 → arduino-cli

```bash
# 编译（分区/PSRAM 都写在命令里，无需手动选）
arduino-cli compile -b esp32:esp32:esp32s3 \
  --board-options PartitionScheme=min_spiffs,PSRAM=opi \
  /path/to/esp32代码

# 网络协议无线上传
arduino-cli upload -b esp32:esp32:esp32s3 \
  -p <ESP_IP> --protocol network \
  --upload-field password=<OTA_PASSWORD> \
  /path/to/esp32代码
```

> 备选（上面两种都不行时再用）：espota.py，随 ESP32 core 附带，
> `python3 espota.py -i <ESP_IP> -p 3232 -a <OTA_PASSWORD> -f 固件.bin`。

> 上传过程中，telnet/串口会打印 `[OTA] 开始无线更新` → `[OTA] 进度 NN%` → `[OTA] 更新完成，设备即将重启`。

## 3. 远程看日志（telnet）

```bash
telnet <ESP_IP> 23
# 或
nc <ESP_IP> 23
```

连上后会实时滚动（全链路联调关键流）：

```
[MQTT] 已连接，已订阅 v5/bag/cmd，已上报 online
[REPORT] temp=24.5 humid=45.0 lat=31.230412 lng=121.473701
[CMD] action=screen_text value=记得带作业本 id=xxxx
[ACK] {"cmd_id":"xxxx","status":0,"msg":"OK"}
[OTA] ...
```

> USB 串口仍显示全部细节日志；telnet 显示联调关键事件流。

---

## 4. MQTT 驱动全链路联调

设备订阅 `v5/bag/cmd`、回 `v5/bag/cmd/ack`（字段 `cmd_id`）。任选一台联网机器：

```bash
# 看设备所有上报
mosquitto_sub -h mqtt.bag.versecraft.cn -p 1883 -t 'v5/bag/#' -v

# 下发屏显文字（屏幕 t_writing/t_replay 会显示，并回 ACK）
mosquitto_pub -h mqtt.bag.versecraft.cn -p 1883 -t v5/bag/cmd \
  -m '{"id":"test-1","action":"screen_text","value":"记得带作业本"}'

# 切专注模式（呼吸蓝灯 + ACK）
mosquitto_pub -h mqtt.bag.versecraft.cn -p 1883 -t v5/bag/cmd \
  -m '{"id":"test-2","action":"mode_switch","value":"focus_mode"}'
```

或直接在网页端 `/interaction` 页用「发送到设备」下发，按对接文档第 8 节走 6 步最小闭环。

## 5. 典型远程联调一轮（Jetson SSH 场景）

你描述的链路：热点开 → Jetson SSH 进去 → 在 Jetson 上跑 Claude Code（有网）。那么一轮是：

1. Jetson 上改 `esp32代码`，`arduino-cli compile`。
2. `arduino-cli upload -p <ESP_IP> --protocol network ...`（或 espota.py）无线刷。
3. `nc <ESP_IP> 23` 看设备重启后日志，确认 `[MQTT] 已连接`。
4. `mosquitto_pub ... v5/bag/cmd ...` 下发命令，telnet 里看 `[CMD]`→`[ACK]`，屏幕/灯有反应。
5. 网页端确认设备在线、传感器/GPS 刷新。

全程零插线。

## 6. 常见坑

- **无线烧录失败/搜不到网络端口**：八成是第一次 USB 烧录时**分区方案没选带 OTA 的**，重新 USB 刷一次。
- **`.local` 解析不了**：直接用 `<ESP_IP>` 代替 `smart-bag-esp32.local`（部分网络不支持 mDNS）。
- **telnet 连不上**：确认设备已 `WiFi 已连接`（无热点时 OTA/telnet 不会启动）；确认同网段、端口 23 未被防火墙挡。
- **OTA 进行中设备卡**：摄像头+大固件时 OTA 期间算力紧张属正常，等进度走完自动重启即可。
- **改了 WiFi SSID/密码**：在 `esp32代码/MQTT.cpp` 顶部 `WIFI_SSID/WIFI_PASS`，这次必须 USB 重刷（因为要先能联网才能 OTA）。
